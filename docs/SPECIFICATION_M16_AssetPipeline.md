# Specification: Milestone 16 - Asset Pipeline & Sprite Management

## Overview
In Milestone 16, "Incredible Machines" will upgrade its rendering system from primitive Pygame geometry (polygons, circles, lines) to a professional sprite-based architecture using PNG image files. To ensure rapid prototyping and prevent crashes from missing textures, the system will implement a robust "auto-generation fallback." If an entity or UI element requests an image that does not exist, the Asset Manager will automatically generate a placeholder image, save it to the hard drive, and use it, allowing artists to easily replace these placeholders later.

## Goals
1. Establish a structured asset directory (`assets/sprites/` and `assets/icons/`).
2. Centralize image loading to avoid frame-rate drops from redundant disk I/O.
3. Implement an auto-generation fallback that creates and saves missing PNG files.
4. Transition in-game entities from primitive rendering to sprite rendering mapped to Pymunk physical bodies.
5. Transition UI buttons from procedurally drawn icons to static loaded icons.

## Technical Implementation Details

### 1. Asset Directory Structure
*   Create `assets/sprites/` to store textures for objects simulated in the physics world (e.g., `Cannon.png`, `Bouncy Ball.png`).
*   Create `assets/icons/` to store explicitly defined UI textures used in the sidebars (e.g., `Cannon_button.png`, `Bouncy Ball_button.png`).

### 2. The Asset Manager (`utils/asset_manager.py`)
*   Develop a centralized `AssetManager` singleton or module.
*   **Cache:** Maintain a dictionary mapping file paths to loaded `pygame.Surface` instances. When an image is requested, check the cache first.
*   **Loading:** If not in the cache, attempt to load the file from disk using `pygame.image.load()`. If successful, cache and return the `Surface`.

### 3. Auto-Generation Logic
*   **Fallback Trigger:** If `pygame.image.load()` raises a `FileNotFoundError`, the auto-generation sequence begins.
*   **Surface Creation:** Create a new `pygame.Surface` of a predetermined size (e.g., matching the entity's config dimensions for sprites, or 60x60 for icons). Fill it with a visible placeholder color (e.g., gray or magenta).
*   **Visual Indicator:** Use `pygame.font` or `pygame.draw` to map text onto the surface, specifically the `entity_id` or requested filename, to clearly identify the missing asset in-game.
*   **Disk Writing:** Use `pygame.image.save()` to write this newly generated generic surface to the hard drive at the originally requested missing path.
*   **Return:** Cache and return the generated `Surface` to the caller.

### 4. Entity Sprite Integration (World Canvas)
*   **Initialization Update:** Modify the base Entity class (e.g., `GamePart.__init__`) or specific subclasses. During initialization, query the `AssetManager` for `assets/sprites/{entity_id}.png`.
*   **Render Update:** Modify the entity's `draw()` or `update_visual()` method. Instead of iterating over Pymunk shapes and drawing primitive Pygame polygons/circles, use `surface.blit()` to render the cached sprite.
*   **Rotation Logic:** The Pymunk `Shape` remains the source of truth for collision. The entity's Pygame render loop must calculate the physical body's current rotation and position, rotate the newly integrated sprite surface, calculate the new bounding-box center, and `blit` it accurately over the physics body.

### 5. UI Icon Integration (Toolbars)
*   **Button Modification:** Update `utils/ui_manager.py` (and relevant calls in `main.py`). Currently, UI buttons are generated by passing a tiny primitive-drawing surface into the constructor.
*   **Icon Fetching:** Change the toolbar initialization loop to request `assets/icons/{entity_id}_button.png` from the `AssetManager`.
*   **Rendering:** Pass this loaded or generated fallback icon `Surface` directly into the `UIButton` for rendering.

## Acceptance Criteria
- [ ] The `assets/sprites/` and `assets/icons/` directories are successfully created and mounted to the project.
- [ ] Spawning an entity whose PNG file does NOT exist results in a placeholder image instantly appearing in-game.
- [ ] The placeholder image is verified to have been saved to the hard drive in the correct `assets/` subdirectory.
- [ ] Replacing the placeholder PNG file on disk and rebooting the game results in the new custom art rendering correctly on the entity.
- [ ] While playing the game, entities correctly spin and rotate their sprites to precisely match their underlying invisible Pymunk collision boxes.
